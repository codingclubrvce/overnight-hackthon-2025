# Round 1 - Detailed Judging Criteria

**Total: 100 Points**

## 1. Problem Understanding & Impact (15 Points)

-   Clear understanding of the root cause and stakeholders affected
-   Identification of why existing solutions fail or are inadequate
-   Recognition of cultural, economic, or regional nuances specific to India
-   Realistic assessment of how many people/organizations benefit

## 2. Innovation & Creativity (15 Points)

-   Originality of the solution compared to existing alternatives
-   Creative use of technology to address the problem uniquely
-   Innovative combination of multiple technologies or methodologies
-   Fresh perspective that challenges conventional thinking

## 3. Technical Implementation & Code Quality (25 Points)

-   Code quality: Clean, well-structured, and maintainable code
-   Appropriate design patterns and architectural decisions
-   Proper error handling and edge case management
-   Core features are working and demonstrable
-   Code readability with proper naming conventions and comments
-   Complexity of algorithms, models, or systems built

## 4. Repository Structure & Documentation (15 Points)

-   Well-organized repository structure with clear folder hierarchy
-   Comprehensive README with setup instructions and dependencies
-   Clear documentation of API endpoints, functions, or modules
-   Commit history showing steady progress and meaningful commit messages
-   Presence of configuration files, environment setup guides

## 5. Hosted Link / Video Demo (20 Points)

-   Code can be installed and run without issues
-   Core functionality works as described in documentation
-   Realistic test cases and data used in demonstration
-   Smooth execution without major technical failures
-   Clear walkthrough of key features

> **Important:** A hosted website link or video demo is compulsory for your submission.

## 6. Feasibility & Practicality (10 Points)

-   Realistic plan for data acquisition and processing
-   Consideration of privacy, security, and compliance requirements
-   Solution doesn't require prohibitively expensive infrastructure
-   Use of open-source or accessible technologies where appropriate
